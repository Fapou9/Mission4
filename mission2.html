Mission 2


L'objectif de cette mission est de débuter l'exploitation de la base de données de l'entreprise KDou afin de permettre, à terme, la production d'analyses et d'indicateurs pertinents. L'utilisation du logiciel Low Code Access est prévue pour la réalisation des requêtes nécessaires.


Le premier indicateur intéressant est le "Quelles sont les produits de type viande livrés à Paris ?".
Cet indicateur est intéressant car nous pouvons voir quels genres de produit sont livré à Paris, ça peut également nous donner une idée sur les produits les plus populaire consommé à la capitale.


Résultat :



SQL :


SELECT DISTINCT Produit.NomProd, Client.VilleCli, Categorie.NomCateg
FROM Client INNER JOIN Commande
ON Client.CodeCli = Commande.CodeCli, Categorie INNER JOIN Produit ON Categorie.CodeCateg = Produit.CodeCateg
WHERE (((Client.VilleCli)="Paris") AND ((Categorie.NomCateg)="Viandes"));


Le deuxième indicateur intéressant est le "Combien y a-t-il de clients nantais ? ".
Cet indicateur est intéressant car c'est important pour nous de savoir notre nombre de clients par villes, ça peut également nous donner une idée sur les villes avec le plus de clients et pouvoir ainsi exploiter ce nombre.


Résultat :



SQL :

SELECT Count(Client.VilleCli) AS CompteDeVilleCli
FROM Client
WHERE (((Client.VilleCli)="Nantes"));


Le troisième indicateur intéressant est le "Donner le prix moyen de vente des produits, le prix minimum et le prix maximum.".
Cet indicateur est intéressant car connaitre ses prix permet de mieux gerer les ventes. Graçe à cette indicateur nous pouvons déduire si les produits coute trop cher ou au contraire pas assez.


Résultat :



SQL :

SELECT Min(Produit.PrixUnit) AS [Min], Avg(Produit.PrixUnit) AS Moyen, Max(Produit.PrixUnit) AS [Max]
FROM Produit;


Le dernier indicateur intéressant est : "Dans quelles villes la clientèle est-elle la plus importante ?".
Cet indicateur est intéressant car connaitre les villes avec le plus de clients permet de mieux gerer ses boutiques et les faire évoluer en fonction pour améliorer l'éxperience du client.


Résultat :



SQL :

SELECT TOP 3 Client.VilleCli, Count(Client.VilleCli) AS Ville
FROM Client
GROUP BY Client.VilleCli
ORDER BY Count(Client.VilleCli) DESC;
